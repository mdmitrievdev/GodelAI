* Original task description:

I'm a full stack developer mostly working with .NET, Angular/React (Type Script), SQL/NoSQL, etc.
There is GitHub Copilot practical task which I should complete during several days (max 2 hr per day):
The goal: deliver a working application where at least 90% of the code is generated by AI tools and you act mainly as a viewer. The idea is up to you. It can be a small UI plus API app but it should include at least:
1. Several pages and components;
2. Routing;
3. A few different layouts (like main layout, admin layout, etc.);

What you must deliver:
1. Working project
	a. A runnable solution (UI + optional API)
	b. Code is mostly generated with Copilot with your review and minimal manual editing
2. Prompt and workflow log (documentation)
	a. A short document describing how you used AI to build the app:
		a.1 All key prompts you used, in order (initial, refined, fixes, refactors, tests, etc.)
		a.2 The context you provided (project description, containers, existing code)
		a.3 The model/tools you used (e.g. GitHub Copilot in Visual Studio/Visual Studio Code, ChatGPT, etc)
		a.4 Any used MCP servers or external tools (what for and how)
	b. For each important step, show:
		b.1 The prompt you used
		b.2 The result summary, not full code
		b.3 What you accepted/changed, and why
3. Insights
	a. First short section with your observations and learnings:
		a.1 Which prompts worked well?
		a.2 Which prompts did not work and why?
		a.3 What patterns in prompting gave the best results (like first ask for architecture, then ask for implementation profile, etc.)?
		
Format
1. Project: In a repository (solution should be build and run). 
2. Documentation: Readme.md (or similar) in the repo with:
	a. Overview of the app
	b. Prompt history (key steps)
	c. Tools/models/MCP used
- Insights and recommendations for future use of Copilot

Project ideas for the course:
1. Skill Extraction tool
An app that reads a new employee's CV and IFU, extracts skills and saves them in a structured format.
2. Test Checking Assistant
A system that checks paper tests, gives feedback based on mistakes, and creates a list of written materials for improvement.
3. Legal document comparator
A tool for lawyers: It reads new versions of legal documents, compares them with old ones, and sends a summary with key changes and their meaning.
4. Performance Review Reminder Bot
A bot that checks who has performance reviews this month, sends reminders to team members, and reports missing feedback to the team manager.
5. Codebase Technology Scanner
A bot that scans a project folder, finds old tools and frameworks with versions, and shows what technologies are active or not. LLMs help extract this information.
